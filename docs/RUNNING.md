# Running Guide

Step-by-step instructions to set up the environment and run the lattice scattering analysis pipeline.

---

## 1. Prerequisites

| Requirement | Version / Notes |
|-------------|-----------------|
| **Python** | 3.10 or newer |
| **pip** | Latest recommended |
| **TeX** | Required for LaTeX plot labels (`text.usetex=True`) |

### TeX installation

| OS | Install |
|----|---------|
| macOS | [MacTeX](https://tug.org/mactex/) or `brew install --cask mactex-no-gui` |
| Linux | `sudo apt install texlive-full` (Debian/Ubuntu) or equivalent |
| Windows | [MiKTeX](https://miktex.org/) or TeX Live |

To disable LaTeX rendering (e.g. for quick testing), set `"text.usetex": False` in `plotting/plot_set.py` → `RC_PARAMS`, or call `apply_plot_style()` after overriding `RC_PARAMS`.

---

## 2. Environment Setup

### Clone / enter the project

```bash
cd /path/to/lattice_scattering
```

### Create a virtual environment (recommended)

```bash
python3 -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows
```

### Install dependencies

```bash
pip install -r requirements.txt
```

### Verify installation

```bash
python -c "import numpy, scipy, matplotlib, gvar, lsqfit, joblib; print('All packages OK')"
```

## 3. Data Preparation

The pipeline expects data under `data/<system>/`. The repository includes input configuration files for `Tcccc6600`, `X3872`, and `Zc3900`, but raw and resampled `.npy` data are **not** synchronized to GitHub. Place the data locally before running.

| System | Input file | Local data directory | Configured ensembles |
|--------|------------|----------------------|----------------------|
| `Tcccc6600` | `input/input_Tcccc6600.py` | `data/Tcccc6600/` | `L12M420_EV170`, `L16M420_EV120` |
| `X3872` | `input/input_X3872.py` | `data/X3872/` | `L16M420_EV70` |
| `Zc3900` | `input/input_Zc3900.py` | `data/Zc3900/` | `L16M420_EV70` |

The directory name must match the string passed to `BuildConfig("<System>")`; for example `BuildConfig("Zc3900")` reads from `data/Zc3900/`.

For Tcccc6600:

```
data/Tcccc6600/
├── raw/
│   ├── correlation_meson_L12M420_EV170.npy
│   ├── correlation_tetraquark_L12M420_EV170.npy
│   ├── correlation_meson_L16M420_EV120.npy
│   └── correlation_tetraquark_L16M420_EV120.npy
└── resampled/          # required for scattering analysis
    ├── resample_En_meson_L12M420_EV170.npy
    ├── resample_En_tetraquark_L12M420_EV170.npy
    ├── resample_ksi_meson_L12M420_EV170.npy
    ├── resample_En_meson_L16M420_EV120.npy
    ├── resample_En_tetraquark_L16M420_EV120.npy
    └── resample_ksi_meson_L16M420_EV120.npy
```

| Directory | Content | Shape |
|-----------|---------|-------|
| `raw/` | Monte Carlo correlation functions | meson: `[chan, mom, t, sample]`; tetraquark: `[chan_src, mom_src, chan_snk, mom_snk, t, sample]` — `mom` = \(n^2\) index |
| `resampled/` | Per-configuration energies and \(\xi\) from jackknife/bootstrap | Generated by `main.py` with `run_resample_analysis=True` (see §5) |

Tetraquark raw-data analysis loads tetraquark raw correlators; meson raw-data analysis loads meson raw correlators. Scattering can be run independently from existing resampled files and requires meson `resample_En_*`, tetraquark `resample_En_*`, and `resample_ksi_meson_*` files for every spatial size **key** in `scattering_Ns_mom` (and in `scattering_Ns_mom_MF` when `run_MF_analysis=True`). Moving-frame scattering additionally reads `resample_En_d{d_x}{d_y}{d_z}_tetraquark_*` (e.g. `d001` for `MF_d_vec=(0, 0, 1)`) when `run_MF_analysis=True`.

The zeta lookup table `data/zeta/zeta_00_rest_lam{λ}_nq{n_q}.npy` is auto-loaded if present; otherwise it is generated on first run (slow — uses `joblib` parallel summation). Default rest-frame cache: `zeta_00_rest_lam50_nq100000.npy`.

### X3872 and Zc3900 data

For `X3872` and `Zc3900`, use the same naming convention with their configured ensemble tag:

```
data/X3872/
├── raw/
│   ├── correlation_meson_L16M420_EV70.npy
│   └── correlation_tetraquark_L16M420_EV70.npy
└── resampled/
    ├── resample_En_meson_L16M420_EV70.npy
    ├── resample_En_tetraquark_L16M420_EV70.npy
    └── resample_ksi_meson_L16M420_EV70.npy

data/Zc3900/
├── raw/
│   ├── correlation_meson_L16M420_EV70.npy
│   └── correlation_tetraquark_L16M420_EV70.npy
└── resampled/
    ├── resample_En_meson_L16M420_EV70.npy
    ├── resample_En_tetraquark_L16M420_EV70.npy
    └── resample_ksi_meson_L16M420_EV70.npy
```

All `data/**/*.npy` files are ignored by `.gitignore`, so these raw and resampled data files remain local and will not be uploaded to GitHub.

## 4. Configuration

Each physical system has an input module under `input/`:

| File | Role |
|------|------|
| `input/config.py` | `InputControlBase`, `BuildConfig`, runtime `Config`, ensemble helpers |
| `input/input_<System>.py` | System overrides + `ENSEMBLE_DATABASE` |

Edit the selected system file before running, e.g. `input/input_Tcccc6600.py`.
Subclass `InputControlBase` and only override fields that differ; the rest inherit defaults.

### Common switches

Set **Effective mass** and **Scattering** blocks in each ``input/input_<Name>.py``.

```python
@dataclass
class InputControl(InputControlBase):
  # --- Effective mass (GEVP, fits, ratio, t_min, resample) ---
  lattice_Ns: int = 16
  run_meson_analysis: bool = False
  run_dispersion_analysis: bool = False      # meson branch only
  run_weight_analysis: bool = False          # meson branch only (Z_n/Z_0)

  run_tetraquark_analysis: bool = True
  run_GEVP_analysis: bool = True          # tetraquark branch only

  run_tmin_analysis: bool = False
  run_ratio_analysis: bool = False
  is_ratio_shift: bool = True       # ratio method: shift vs C4/(C_a C_b)

  run_resample_analysis: bool = False        # last: jackknife/bootstrap → resampled/

  # --- Scattering (K(s), kcot from resampled En) ---
  run_scattering_analysis: bool = True
  scattering_channel: str = r"J/\psi\,J/\psi"
  scattering_Ns_mom: Dict[int, List[int]] = {12: [0, 1, 2], 16: [0, 1]}
  run_MF_analysis: bool = False
  # ... scattering_channel_MF, scattering_Ns_mom_MF, zeta, plot ranges
```

**Effective mass** defaults (``InputControlBase``): ``t_run_*``, ``ratio_at``, ``resample_type``, ``plot_format``.

**Scattering** defaults (``InputControlBase``): zeta tables, ``scattering_fit_mode``, axis ranges, ``MF_d_vec``.

| Switch | Section | Effect |
|--------|---------|--------|
| `run_tmin_analysis` | Effective mass | Scan `t_min`: meson 2-state \(E\); tetraquark 3-state + ratio |
| `run_ratio_analysis` | Effective mass | Ratio fits (`R_n` or `R=C₄/CₐCᵦ`) and `Ratio_*` figures |
| `t_run_start` / `t_run_stop` / `t_run_step` | Effective mass | Grid of scanned `t_min` values |
| `ratio_at` | Effective mass | Offset ``a_t`` in ``R_n(t+a_t)`` |
| `lattice_Ns` | Effective mass | Primary ensemble for raw-data analysis |
| `run_tetraquark_analysis` | Effective mass | Tetraquark GEVP + 3-state fits + figures |
| `run_meson_analysis` | Effective mass | Meson 2-state fits; `En` figures |
| `run_dispersion_analysis` | Effective mass | Meson dispersion fit + figure (meson branch only) |
| `run_weight_analysis` | Effective mass | Meson \(Z_n/Z_0\) figure (meson branch only) |
| `run_GEVP_analysis` | Effective mass | GEVP diagonalization and matrix plots (tetraquark branch only) |
| `is_ratio_shift` | Effective mass | Ratio method: shift vs direct quotient |
| `run_resample_analysis` | Effective mass | Write jackknife/bootstrap `resampled/` (no figures) |
| `scattering_channel` | Scattering | Tetraquark label for \(K(s)\) (`input/input_*.py`; runtime `Config.scattering_chan`) |
| `scattering_channel_MF` | Scattering | Moving-frame tetraquark label (runtime `Config.scattering_chan_MF`) |
| `scattering_Ns_mom` | Scattering | Spatial \(L\) keys and momentum indices per \(L\) |
| `scattering_Ns_mom_MF` | Scattering | Moving-frame momentum indices per \(L\) |
| `scattering_fit_mode` | Scattering | `"Ks_linear"` or `"kcot_quadratic"` |
| `rest_zeta_lamda` | Scattering | Rest-frame zeta cutoff \(\lambda\) |
| `rest_zeta_n_q` | Scattering | Rest-frame zeta \(q^2\) grid size |
| `regen_rest_zeta` | Scattering | Rebuild rest-frame zeta cache |
| `MF_zeta_lamda` | Scattering | Moving-frame zeta cutoff |
| `MF_zeta_n_q` | Scattering | Moving-frame zeta grid size |
| `regen_MF_zeta` | Scattering | Rebuild moving-frame zeta cache |
| `regen_MF_scattering` | Scattering | Recompute moving-frame per-sample scattering |
| `k_sq_plot_range` | Scattering | kcot axes `(k^2_lo, k^2_hi, y_lo, y_hi)` |
| `s_plot_range` | Scattering | K(s) axes `(s_lo, s_hi, y_lo, y_hi)` |
| `MF_d_vec` | Scattering | Boost direction \(\mathbf d\) on lattice |
| `run_scattering_analysis` | Scattering | \(K_s\) / \(k\cot\delta\) from resampled files + figures |
| `run_MF_analysis` | Scattering | Include moving-frame tetraquark resamples |
| `plot_format` | both | Figure format `"png"` or `"pdf"` |

**Analysis switches:** `run_meson_analysis` and `run_tetraquark_analysis` are independent. Set both to `False` to skip raw-data analysis and run scattering from existing resampled files.

To switch the physics system, change the name in `main.py`:

```python
builder = BuildConfig("Tcccc6600")  # or "X3872", "Zc3900"
```

## 5. Running the Pipeline

Always run from the **project root** so relative paths (`data/`, `result/`) resolve correctly.

### Resampling (generate `resampled/` files)

Scattering analysis requires jackknife/bootstrap outputs. To generate them, enable the desired raw-data analysis branch and set `run_resample_analysis=True`:

```bash
python main.py
```

Configure the selected `input/input_<System>.py`:

- `resample_type`: `"jackknife"` or `"bootstrap"`
- `lattice_Ns`: ensemble to resample (run once per volume if needed)
- `run_meson_analysis` / `run_tetraquark_analysis`: analysis type for this run
- `run_resample_analysis=True`: write `data/<system>/resampled/*.npy`
- `run_scattering_analysis=False`: optional while generating resampled files only

**For full scattering**, generate all required files:

| Run | Settings | Output |
|-----|----------|--------|
| Meson L12 | `lattice_Ns=12`, `run_meson_analysis=True` | `resample_ksi_meson_L12M420_EV170.npy`, `resample_En_meson_...` |
| Meson L16 | `lattice_Ns=16`, `run_meson_analysis=True` | `resample_ksi_meson_L16M420_EV120.npy`, ... |
| Tetraquark L12 | `lattice_Ns=12`, `run_tetraquark_analysis=True` | `resample_En_tetraquark_L12M420_EV170.npy` |
| Tetraquark L16 | `lattice_Ns=16`, `run_tetraquark_analysis=True` | `resample_En_tetraquark_L16M420_EV120.npy` |

### Main analysis and scattering

```bash
python main.py
```

Default execution order:

1. Build configs from `input/input_<System>.py`
2. If `run_resample_analysis=True`, generate resampled files for each enabled analysis branch
3. If `run_resample_analysis=False` and `run_meson_analysis=True`, load meson raw correlators, fit, and plot meson observables
4. If `run_resample_analysis=False` and `run_tetraquark_analysis=True`, load tetraquark raw correlators, run GEVP/fits, and plot tetraquark observables
5. If `run_tmin_analysis=True`, scan `t_min` and save `En_tmin_meson{n}_mom{m}_*` / `En_tmin_tetraquark{n}_mom{m}_*` figures
6. If `run_scattering_analysis=True`, load existing resampled `.npy` files and run scattering analysis → `K_s_scattering.{png|pdf}`, `kcot_scattering.{png|pdf}`

To run scattering only from resampled files, set `run_meson_analysis=False`, `run_tetraquark_analysis=False`, and `run_scattering_analysis=True`.

### \(t_{\min}\) scan and ratio plots

Enable in `input/input_Tcccc6600.py` (defaults shown):

```python
run_tmin_analysis: bool = True
run_ratio_analysis: bool = True
run_meson_analysis: bool = False     # optional full meson branch; ratio t_min only needs run_ratio_analysis
run_tetraquark_analysis: bool = True
```

`main.py` runs the meson branch first (fits only; t_min plotting skipped on meson), then the tetraquark branch with meson correlators available for ratio. Output examples on \(L=16\):

```
result/Tcccc6600/
├── En_tmin_meson0_mom0_L16M420_EV120.png
├── En_tmin_tetraquark1_mom0_L16M420_EV120.png
├── En_tmin_tetraquark1_mom1_L16M420_EV120.png
└── ...
```

`E2` = second tetraquark chan (\(J/\psi\,J/\psi\)); `mom{k}` = \(n^2=k\). Y-axis is energy in GeV; vertical range is ±10× the Combine reference error bar.

### Headless / server (no display)

If running on a cluster without a GUI, set the matplotlib backend before importing plotting modules. Add at the top of `main.py`:

```python
import matplotlib
matplotlib.use("Agg")
```

Plots will still be saved to `result/`; `plt.show()` will not open a window.


## 8. Troubleshooting

| Problem | Likely cause | Fix |
|---------|--------------|-----|
| `FileNotFoundError: Missing file: data/...` | Required local `.npy` not present | Place raw files in `data/<system>/raw/` or resampled files in `data/<system>/resampled/` with correct naming |
| Scattering step returns `None` | `run_scattering_analysis=False` | Set `run_scattering_analysis=True` |
| Missing resampled files | Resampling has not been run for all required ensembles | Set `run_resample_analysis=True` and run `main.py` for each required branch/volume |
| LaTeX error on plot | TeX not installed | Install TeX or set `"text.usetex": False` in `plotting/plot_set.py` → `RC_PARAMS` |
| `Unknown lattice_Ns` | Invalid `lattice_Ns` value | Use `12` or `16` (see `get_lattice_params`) |
| `Unknown ensemble key` | `lattice_Ns` / analysis type mismatch | Check `ENSEMBLE_DATABASE` keys in input file |
| Slow first run | Zeta table generation | Wait for completion; file is cached under `data/zeta/zeta_00_rest_lam{λ}_nq{n_q}.npy` |
| Memory / time on resample | Full jackknife over all configs | Expected; reduce test size or run on a machine with more RAM |

## 9. Adding a New Physics System

1. Copy `input/input_Tcccc6600.py` → `input/input_<NewSystem>.py`
2. Fill in `InputControl` defaults and `ENSEMBLE_DATABASE` entries
3. Place correlators under `data/<NewSystem>/raw/`
4. Update `main.py`:

   ```python
   builder = BuildConfig("<NewSystem>")
   ```
