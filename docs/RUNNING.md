# Running Guide

Step-by-step instructions to set up the environment and run the lattice scattering analysis pipeline.

---

## 1. Prerequisites

| Requirement | Version / Notes |
|-------------|-----------------|
| **Python** | 3.10 or newer |
| **pip** | Latest recommended |
| **TeX** | Required for LaTeX plot labels (`text.usetex=True`) |

### TeX installation

| OS | Install |
|----|---------|
| macOS | [MacTeX](https://tug.org/mactex/) or `brew install --cask mactex-no-gui` |
| Linux | `sudo apt install texlive-full` (Debian/Ubuntu) or equivalent |
| Windows | [MiKTeX](https://miktex.org/) or TeX Live |

To disable LaTeX rendering (e.g. for quick testing), set `"text.usetex": False` in `plotting/plot_set.py` â†’ `RC_PARAMS`, or call `apply_plot_style()` after overriding `RC_PARAMS`.

## 3. Data Preparation

The pipeline expects data under `data/<system>/`. The repository includes input configuration files for `Tcccc6600`, `X3872`, and `Zc3900`, but raw and resampled `.npy` data are **not** synchronized to GitHub. Place the data locally before running.

| System | Input file | Local data directory | Configured ensembles |
|--------|------------|----------------------|----------------------|
| `Tcccc6600` | `input/input_Tcccc6600.py` | `data/Tcccc6600/` | `L12M420_EV170`, `L16M420_EV120` |
| `X3872` | `input/input_X3872.py` | `data/X3872/` | `L16M420_EV70` |
| `Zc3900` | `input/input_Zc3900.py` | `data/Zc3900/` | `L16M420_EV70` |

## 4. Configuration

Edit the selected system input file before running, e.g. `input/input_Tcccc6600.py`, `input/input_X3872.py`, or `input/input_Zc3900.py`.

### Common switches

```python
@dataclass
class InputControl:
    lattice_Ns: int = 12              # primary ensemble: 12 or 16
    is_meson_analysis: bool = False   # meson Zn / dispersion runs
    is_tetraquark_analysis: bool = True  # GEVP + tetraquark raw-data analysis
    is_gevp: bool = True              # GEVP diagonalization
    run_scattering: bool = True       # scattering Ks / kcot analysis
    run_resample: bool = False        # generate resampled files from raw data
    is_moving_frame: bool = False               # include moving-frame tetraquark energies
    plot_meff: bool = True            # En and Zn plots
    plot_dispersion: bool = True      # dispersion fit + plot (meson only)
    plot_format: str = "png"          # figure output: "png" or "pdf"
    resample_type: str = "jackknife"  # or "bootstrap"

    # Scattering channel indices and K(s) fit momentum subsets
    ch_meson_a: int = 1
    ch_meson_b: int = 1
    ch_tetra: int = 1
    ch_tetra_MF: int = 0
    fit_mom_by_ns: Dict[int, List[int]] = {12: [0, 1, 2], 16: [0, 1]}
    fit_mom_by_ns_MF: Dict[int, List[int]] = {16: [0, 1]}
    scattering_fit_mode: str = "Ks_linear"  # see analysis/scattering.py (SCATTERING_FIT_MODES)
    rest_zeta_lamda: int = 50
    rest_zeta_n_q: int = 100_000
    regen_rest_zeta: bool = False
    moving_frame_zeta_lamda: int = 50
    moving_frame_zeta_n_q: int = 1000
    regen_moving_frame_zeta: bool = False
    regen_moving_frame_scattering: bool = False
    k_sq_plot_range: tuple[float, float] = (-0.25, 1.75)
    s_plot_range: tuple[float, float] = (37, 45)
    moving_frame_d_vec: tuple[int, int, int] = (0, 0, 1)
```

| Switch | Effect |
|--------|--------|
| `lattice_Ns` | Selects which raw correlator set to load and fit |
| `is_tetraquark_analysis` | Uses tetraquark channels and 3-state cosh model; enables GEVP and scattering |
| `is_meson_analysis` | Uses meson channels and 2-state cosh model; enables `Zn` and dispersion plots |
| `is_gevp` | Solve GEVP and produce GEVP matrix / eigenvector plots (tetraquark mode) |
| `ch_meson_a` / `ch_meson_b` / `ch_tetra` | Channel indices used in scattering analysis |
| `ch_tetra_MF` | Moving-frame tetraquark channel index; ignored when `is_moving_frame=False` |
| `fit_mom_by_ns` | Momentum subsets per \(L\) for the \(K(s)\) linear fit |
| `fit_mom_by_ns_MF` | Moving-frame levels per \(L\) for the \(K(s)\) linear fit; ignored when `is_moving_frame=False` |
| `scattering_fit_mode` | Scattering fit in `analysis/scattering.py`: `"Ks_linear"` fits \(K(s)\) linearly in \(s\); `"kcot_quadratic"` fits \(k\cot\delta_0=a(k^2)^2+bk^2+c\) |
| `rest_zeta_lamda` | Cutoff \(\lambda\) for rest-frame zeta summation |
| `rest_zeta_n_q` | Number of \(q^2\) grid points for rest-frame zeta table |
| `regen_rest_zeta` | Rebuild cached rest-frame zeta tables in `data/zeta/` |
| `moving_frame_zeta_lamda` | Cutoff \(\lambda\) for moving-frame zeta summation; ignored when `is_moving_frame=False` |
| `moving_frame_zeta_n_q` | Number of \(q^2\) grid points for moving-frame zeta table; ignored when `is_moving_frame=False` |
| `regen_moving_frame_zeta` | Rebuild cached moving-frame zeta tables in `data/zeta/` |
| `regen_moving_frame_scattering` | Recompute moving-frame per-sample scattering; otherwise load `resample_scatter_MF_*.npy` from `resampled/` |
| `k_sq_plot_range` | Horizontal plotting range for the \(k\cot\delta_0\) figure |
| `s_plot_range` | Horizontal plotting range for the \(K(s)\) figure |
| `moving_frame_d_vec` | Integer total-momentum vector \(\mathbf d\), with \(\mathbf P=2\pi\mathbf d/L\) |
| `run_resample` | Generate jackknife/bootstrap `resampled/` files from enabled raw-data analyses |
| `run_scattering` | Computes \(K_s\) and \(k\cot\delta\) from existing `resampled/` files |
| `is_moving_frame` | If `True`, also loads `resample_En_d{d_x}{d_y}{d_z}_tetraquark_*` and uses moving-frame kinematics |
| `plot_meff` | Plot effective-mass \(E_n\) and (for meson runs) \(Z_n/Z_0\) |
| `plot_dispersion` | Fit and plot dispersion relation (meson mode only) |
| `plot_format` | Output figure format: `"png"` (default) or `"pdf"` |

**Analysis switches:** `is_meson_analysis` and `is_tetraquark_analysis` are independent. Set both to `False` to skip raw-data analysis and run scattering from existing resampled files.

To switch the physics system, change the name in `main.py`:

```python
builder = BuildConfig("Tcccc6600")  # or "X3872", "Zc3900"
```

## 5. Running the Pipeline

Always run from the **project root** so relative paths (`data/`, `result/`) resolve correctly.

### Resampling (generate `resampled/` files)

Scattering analysis requires jackknife/bootstrap outputs. To generate them, enable the desired raw-data analysis branch and set `run_resample=True`:

```bash
python main.py
```

Configure the selected `input/input_<System>.py`:

- `resample_type`: `"jackknife"` or `"bootstrap"`
- `lattice_Ns`: ensemble to resample (run once per volume if needed)
- `is_meson_analysis` / `is_tetraquark_analysis`: analysis type for this run
- `run_resample=True`: write `data/<system>/resampled/*.npy`
- `run_scattering=False`: optional while generating resampled files only

**For full scattering**, generate all required files:

| Run | Settings | Output |
|-----|----------|--------|
| Meson L12 | `lattice_Ns=12`, `is_meson_analysis=True` | `resample_ksi_meson_L12M420_EV170.npy`, `resample_En_meson_...` |
| Meson L16 | `lattice_Ns=16`, `is_meson_analysis=True` | `resample_ksi_meson_L16M420_EV120.npy`, ... |
| Tetraquark L12 | `lattice_Ns=12`, `is_tetraquark_analysis=True` | `resample_En_tetraquark_L12M420_EV170.npy` |
| Tetraquark L16 | `lattice_Ns=16`, `is_tetraquark_analysis=True` | `resample_En_tetraquark_L16M420_EV120.npy` |

### Main analysis and scattering

```bash
python main.py
```

Default execution order:

1. Build configs from `input/input_<System>.py`
2. If `run_resample=True`, generate resampled files for each enabled analysis branch
3. If `run_resample=False` and `is_meson_analysis=True`, load meson raw correlators, fit, and plot meson observables
4. If `run_resample=False` and `is_tetraquark_analysis=True`, load tetraquark raw correlators, run GEVP/fits, and plot tetraquark observables
5. If `run_scattering=True`, load existing resampled `.npy` files and run scattering analysis â†’ `K_s_scattering.{png|pdf}`, `kcot_scattering.{png|pdf}`

To run scattering only from resampled files, set `is_meson_analysis=False`, `is_tetraquark_analysis=False`, and `run_scattering=True`.

### Headless / server (no display)

If running on a cluster without a GUI, set the matplotlib backend before importing plotting modules. Add at the top of `main.py`:

```python
import matplotlib
matplotlib.use("Agg")
```

Plots will still be saved to `result/`; `plt.show()` will not open a window.

## 7. Troubleshooting

| Problem | Likely cause | Fix |
|---------|--------------|-----|
| `FileNotFoundError: Missing file: data/...` | Required local `.npy` not present | Place raw files in `data/<system>/raw/` or resampled files in `data/<system>/resampled/` with correct naming |
| Scattering step returns `None` | `run_scattering=False` | Set `run_scattering=True` |
| Missing resampled files | Resampling has not been run for all required ensembles | Set `run_resample=True` and run `main.py` for each required branch/volume |
| LaTeX error on plot | TeX not installed | Install TeX or set `"text.usetex": False` in `plotting/plot_set.py` â†’ `RC_PARAMS` |
| `Unknown lattice_Ns` | Invalid `lattice_Ns` value | Use `12` or `16` (see `get_lattice_params`) |
| `Unknown ensemble key` | `lattice_Ns` / analysis type mismatch | Check `ENSEMBLE_DB` keys in input file |
| Slow first run | Zeta table generation | Wait for completion; file is cached under `data/zeta/zeta_00_rest_lam{Î»}_nq{n_q}.npy` |
| Memory / time on resample | Full jackknife over all configs | Expected; reduce test size or run on a machine with more RAM |
